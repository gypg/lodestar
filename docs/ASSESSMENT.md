# Lodestar 评估与差异化规划（ASSESSMENT）

> 应你要求：诚实评估"为什么现在还像 octopus"、缺什么、怎么优化，并给出可执行规划。
> 原则：**能从参考项目（new-api / octopus-lingyu / SAPI）改造复用的，绝不重写。**
> 最后更新：2026-06-18。

## 一、诚实评估：为什么你"看不出和 octopus 的区别"

你的直觉是对的。盘点已改 vs 未改：

**已经差异化的（属于我们）**
- 身份：改名 Lodestar（banner/标题/登录/manifest/key 前缀 `sk-lodestar-`）。
- 每用户主题系统（5 预设 + 可上传 + 绑账户）——octopus 完全没有。
- 冬日封面落地页 + 极光氛围。
- 一键商业开关 + 公开注册。

**没改、所以"像 octopus"的（根因）**
1. **进站后的内部页面 100% 是 octopus**：渠道/分组/模型/日志/告警/运维/设置——UI 和交互原样未动。你一旦登录进去，就是 octopus。
2. **没有"公开平台门面"**：octopus 是纯管理后台，**站点名/公告/模型目录/用量全在登录后**，对访客零展示。所以"进网址"除了封面，看不到任何属于平台的内容。**这是最致命的"没区别"**——一个聚合平台该有对外的脸（公告、能用哪些模型、价格、用量），octopus 一样都没有。
3. **站点身份不可配置**：没有"站点名称/简介/公告"这种平台级设置，无法体现"这是某某的聚合站"。

**结论**：差异化不该靠重写内部管理页（那是 octopus 的价值，重写不智），而该靠**给它一个 octopus 没有的"公开平台层"**——这正是 new-api 有、octopus 没有的东西，也是你一直要的首页。

## 二、缺什么（按优先级）

| # | 缺口 | 来源参考 | 价值 |
|---|------|----------|------|
| **P1** | **公开平台门面**：进站封面 + 左侧导航 + 公开展示〔站点公告 / 模型广场(名称+价格) / 用量概览"此刻的数字" / 关于本站〕，私密项(控制台/密钥)点击才要登录 | new-api 首页模式 + octopus 数据层(llm.List/stats) | ★★★ 最大差异化，正是你要的首页 |
| **P2** | **站点身份可配置**：站点名称/简介/公告/页脚 设置项，落地页与全站引用 | new-api 的 SystemName/Notice 模式 | ★★★ 让平台"属于自己"、可运营 |
| P3 | **模型广场公开版**：访客可看支持的模型与价格（不登录） | new-api pricing / SAPI 模型页 | ★★ 聚合平台标配 |
| P4 | 内部页面品牌化微调（配色已随主题；可加 logo、页脚署名、空状态文案） | — | ★ 减少"octopus 感" |
| P5 | 商业纵深（计费/订阅/配额）——开放注册已就位 | new-api subscription / SAPI billing | ★★ 商业化时做 |

## 三、规划（本轮执行 P1+P2+P3，复用为主）

### 后端（复用 octopus 数据层 + 设置系统，只新增薄端点）
1. 新增设置项：`site_name`(默认 Lodestar)、`site_description`、`site_announcement`、`site_footer`。走现成 `DefaultSettings()` + 设置 API。
2. 新增**公开**端点 `GET /api/v1/site/public-overview`（无鉴权）：聚合返回
   `{site_name, description, announcement, footer, model_count, models:[{name,input,output}], total_requests, total_tokens}`。
   复用 `op/llm.List`(模型+价格) + `op/stats.TotalGet`(总量) + 设置。**不暴露任何私密数据**（无 key、无用户、无渠道地址）。

### 前端（落地页升级为信息门面）
3. `WinterLanding` 公开变体升级：左侧目录分两类——
   - **公开项**（公告/模型广场/用量概览/关于本站）：点击在中部/右侧浮出内容面板，**无需登录**即看；数据来自 public-overview。
   - **私密项**（控制台/密钥/渠道…）：点击 → 唤出登录。
4. 站点名/公告接入封面刊头（替换写死的 "Lodestar Daily" → 真实站点名）。
5. 管理端设置面板加"站点信息"卡片（名称/简介/公告/页脚），复用现有 setSetting。

### 验收
- 未登录访客在首页能看到：站点名、公告、支持的模型列表+价格、用量总数、关于；点控制台才要登录。
- 站点名/公告可在设置里改，落地页实时反映。
- 全程复用 octopus 数据层 + new-api 首页交互思路，无重写。

## 四、之后（登记，不在本轮）
- P4 内部页品牌化细节、P5 商业纵深（计费/订阅）、落地页可选实景图+color4bg、模块路径改名、新增 UI 文案抽 i18n。

> 一句话：**差异化 = 给 octopus 装上它没有的"公开聚合平台门面 + 可配置站点身份"，内部管理能力照单全收复用。**
